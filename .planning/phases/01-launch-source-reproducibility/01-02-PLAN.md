## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User shell -> Docker runtime | Untrusted local env/config crosses into the app container |
| `.env` file -> startup scripts and backend config | Secrets are loaded and validated during launch |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-04 | Information Disclosure | `scripts/start_mac.sh`, `scripts/start_windows.ps1`, README docs | high | mitigate, blocking | Never echo secret values; only name missing keys, point to `.env.example`, and stop execution before `docker run` on missing `.env` or `OPENROUTER_API_KEY` |
| T-01-05 | Tampering | `backend/app/llm/chat.py` import-time env loading | medium | mitigate | Remove implicit import-time `load_dotenv()` behavior and validate config at startup/runtime boundary |
| T-01-06 | Elevation of Privilege | alternate contributor workflow becoming primary path | low | accept | Document local contributor mode as secondary only; preserve Docker-first student workflow as the only authoritative path |
| T-01-07 | Integrity | market-source selection | medium | mitigate | Keep `create_market_data_source()` env-driven and test simulator default plus Massive-enabled path explicitly |
</threat_model>
